Ruby Gem Structure

Two main parts of a gem

  • The lib directory - where your code resides
  • The gemspec file - defines the Gem::Specification object which is handled by rubygems to manage your code

Additional stuff

  • Spec directory - For Testing your code of course!
  • Pkg directory - for your gem builds
  • RakeFile - Running rake tasks
  • Other files (README, Binaries, etc)

Lets build a simple gem called Printer

Easiest way is to use Bundler

bundle gem printer


simple right?

Lets go over the code

Build our first spec

              
  require File.expand_path(File.dirname(__FILE__) + '/../lib/printer')

  describe SimplePrint do
    describe ".print" do
      it "should print out the inputted value" do
        input = "foobar"
        SimplePrint.should_receive(:puts).with(input)
        SimplePrint.print(input)
      end
    end
  end

Here is the code

              
                # lib/printer.rb
                require "printer/version"

                class SimplePrint
                  def self.print(input)
                    puts input
                  end
                end

Run the specs

Boom! They are green!

Versioning

Simple Rules


Patch 0.0.x

(e.g. Bug Fixes)


Minor 0.x.0

(e.g. backwards compatible changes (new features))


Major x.0.0

(e.g. incompatible changes (break existing users code))

Actions to take

  1. Change the version number in your version.rb file
  2. Tag your version in Git

Lets build and install our gem!

Build, Install, Deploy

Easy with bundler's Rake tasks

              
                rake -T # see what tasks are available

                rake build # creates the gem in the pkg directory

                rake install # installs the gem locally

                rake release # this will release to ruby gems

Commit and push! Done!
